http메서드를 안붙이고 uri로 요청할 때 기본값이 get이다.

post : 추가

@Controller
화면을 반환한다.

@RestController
데이터를 반환하는 컨트롤러로 사용된다.
JSON이나 XML 형식의 데이터를 반환하는 RESTful API를 제공하는 데 적합하다

@Controller + @ResponseBody의 결합이다.

@ResponseBody
메서드의 반환값을 http response body로 직렬화 해 클라이언트에게 전달한다.
따라서 데이터를 JSON, XML형식으로 변환해서 반환한다.


@GetMapping
스프링 프레임워크에서 HTTP GET  요청을 처리하기 위한 어노테이션이다
특정 URL 경로로 들어오는 GET 요청을 처리하고, 그 요청에 대한 응답을 반환하는데 사용된다.

@PathVariable
- 스프링에서 URL의 경로의 일부를 변수로 사용할 수 있도록 지원하는 어노테이션이다.

RESTful API에서 URL은 어떤 리소스를 요청하는지 표현하는 주소이다.
URL은 명사(자원 이름)위주로 설계하고, 동사(행위)는 http메서드로 표현하는것이 핵심이다.

GET /users
POST /users
GET /users/10 -> id가 10인 사용자의 정보를 주세요

GET /getUsers -> X

계층 구조는 상위 리소스 / 하위 리소스 형태로 표현

GET /users/1/todos -> id가 1인 유저의 할일 목록

리소스를 식별할 때는 @PathVariable을 사용한다.
GET /products/100 -> id가 100인 상품

@GetMapping(/products/{id})
public Product getProduct(@PathVariable Long id) { ... }

@RequestParam
쿼리 파라미터나 폼 데이터를 처리하기 위해 사용하는 어노테이션이다.

GET /users?name=kim

@RequestBody
- 요청 본문에 담긴 데이터를 자바 객체로 변환하여 컨트롤러 메서드의 매개변수로 전달하는데 사용하는 어노테이션
- 클라이언트가 전송한 JSON,XML등을 자바 객체로 변환하는 역할을 한다.

@ResponseBody
반환값을 JSON, XML, 문자열과 같은 데이터를 클라이언트에게 반환한다.

ResponseEntity
스프링 프레임워크에서 제공하는 클래스로 HTTP응답을 세밀하게 제어할 수 있다
HTTP 상태코드, 헤더, Response Body를 포함한 전체 HTTP 응답을 구성할 수 있으며 주로 REST API에서 많이 사용된다.

ResponseEntity.ok() : 200 상태코드로 응답하는 메서드
.status(HttpStatus status) : 특정 상태 코드를 반환하는 메서드
.badRequest() : 400 badRequest을 반환하는 메서드
.notFound() : 404 notFound를 반환하는 메서드

우리가 작성할 컨트롤러에서 데이터를 반환할 때 ResponseEntity를 사용할 예정
