상속
부모의 멤버(필드, 메서드)를 자식클래스가 물려받아서 사용하는 것

부모클래스(넓은 카테고리, 일반적인 개념, 추상적인 개념)
ex) 자동차, 사람


자식클래스(좁은 카테고리, 구체적인개념)
ex) 버스, 학생

공통적인 내용을 부모한테 정의해 놓고 자식클래스는 사용하면 되기 때문에
중복이 적어진다는 장점이 있다.

부모클래스쪽에 있는 메서드는 자식이 오버라이드 해서 활용하기 위해 존재
추상화 -> 추상클래스

추상메서드를 최소 1개 이상 가지고 있는 클래스
abstract 키워드를 가지고 있다.
public abstract class 클래스명{

	추상메서드도 abstract키워드를 가지고 있다.
	추상메서드는 body부분이 없다.
	public abstract void 메서드명(매개변수);
}

추상클래스에는 추상메서드 뿐만 아니라 일반적인 필드나 메서드를 가지는것도 가능하다.

추상클래스는 객체를 직접 만들 수 없고, 추상클래스를 상속받는 자식 클래스가 추상메서드를 반드시 구현(오버라이드)해야 한다.

실무에서의 인터페이스
확장성을 염두해두고 설계할 때 사용하는 경우가 많다.

인터페이스명 : Playable
추상메서드 : void play();

클래스명 : MusicPlayer (Playable 인터페이스 구현)
play() 실행시 "음악 재생" 문자열 출력하기

Main에서 객체 만들어서 play() 호출하고 결과 확인하기

인터페이스명 : Printable
추상메서드 : void print() 정의

인터페이스명 : Sortable
추상메서드 : void sort();

클래스명 : DocumentProcessor
두 인터페이스 모두 구현하기
print() : "문서 출력"
sort() : "문서 정렬"

인터페이스 : MessageSender
- void sned(String msg);

클래스 : EmailSender, SmsSender (인터페이스 구현)
Email : "이메일 발송 <메시지>"
SMS : "SMS 발송 <메시지>"

NotificationService 클래스를 만들고
void notifyUser(MessageSender sender, String message) 메서드 만들기
- 각 sender의 send()를 호출하면 됩니다.

Main에서 EmailSender, SmsSender 테스트하기

인터페이스 : Payment
- void pay(int amount);

클래스 : CardPayment, KakaoPayPayment (Payment 구현하기)
Card : "카드로 xx원 결제"
Kakao : "카카오페이로 xx원 결제"

결제기능을 가지고 있는 PayService 클래스 
public void processPayment(Payment payment, int amount)
