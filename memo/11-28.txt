getter & setter

클래스의 필드에 함부로 접근하지 못하도록 private 접근제한자를 사용한다.

setter
필드의 값을 세팅해주는 메서드
메서드명은 'set필드명'을  주로 사용한다.
유효하지 않은 값이 들어왔을 때 검증도 가능하다.

getter
필드의 값을 반환해주는 메서드
메서드명은 'get필드명'을 주로 사용한다.
boolean 필드는 getter를 쓸 때 'is필드명'을 사용하는 경우가 많다.

필드별로 setter와 getter를 만들어야 한다.

lombok이라는 라이브러리가 존재한다.

@xxx -> 어노테이션 (컴파일러한테 알려주는 주석)

어노테이션을 통해서 기본생성자, 매개변수를 갖는 생성자, getter,setter
자동으로 만들 수 있다.

@Data
@getter : getter메서드를 만들어준다.
@setter : setter 메서드를 만들어준다.
@NoArgsConstructor : 기본생성자를 만들어준다.
@RequiredArgsConstructor : 필요한 필드만 생성자의 매개변수로 만들어준다.
@AllArgsConstructor : 모든 필드를 매개변수로 갖는 생성자를 만들어준다.
@Builder : 객체를 빌더패턴으로 만들 수 있게 해준다.


상속
부모클래스와 자식클래스가 존재

부모클래스에 정의된 멤버를 자식클래스에서 가져다가 쓸 수 있다.
ㄴ언제? 자식클래스의 객체가 만들어 질 때

다중 상속이 가능하다.
하나의 부모는 여러 자식 클래스를 가질 수 있다.
하나의 자식클래스는 여러 부모 클래스를 가질 수 없다.

N차 상속이 가능하다.
Car -> Bus -> SchoolBus

SchoolBus는 Car의 멤버를 사용할 수 있다.

오버라이딩
부모가 가지고 있는 메서드가 자식의 상황과 맞지 않을 때 자식 클래스에서 메서드를 재정의할 수 있다.

시그니처 부분(접근제한자, 반환타입, 메서드명, 매개변수)는 일치하고
body부분(코드블럭 내부)의 내용만 바뀐다.

final클래스
부모 클래스가 final이면 자식이 상속받을 수 없다.

final메서드
부모클래스의 메서드가 final로 지정되면 자식이 오버라이딩 할 수 없다.

생성자에는 final을 추가할 수 없다.

public : 어디서든지 접근이 가능하다
private : 같은 클래스에서만 접근이 가능하다.
default : 같은 패키지에서만 접근이 가능하다.
protected : 다른 패키지에서는 접근이 불가능하다, 상속관계라면 super()를 통해 접근이 가능하다.

클래스의 형변환(타입변환)

자동타입변환
작은 자료형 -> 큰 자료형
부모 클래스(큰자료형)
자식 클래스(작은자료형)

자식 "객체"를 부모 타입의 참조변수에 대입하면 자식 클래스에 정의된 멤버는 사용할 수 없다.

같은 이름의 필드를 정의하면 가려짐 상태(hiding)가 된다.

강제타입변환
큰 자료형 -> 작은 자료형
(데이터타입)변수명;
부모 클래스(큰자료형)
자식 클래스(작은자료형)

(자식클래스)변수명;

자식타입에서 부모 타입으로 변환을 했지만 자식 클래스의 멤버에 접근하고 싶을 때 다시 자식타입으로 변경해서 접근할 수 있도록 해야 한다.

"자식 객체가 부모 타입의 변수에 들어있는데", 다시 자식타입으로 변환하는 것

일회성으로 타입 변환이 필요할 때

((자식클래스명)변수명).메서드명();

아예 다시 자식타입으로 바꿀 때

변수명 = (자식클래스명)부모타입;

다형성
사용방법은 동일하지만 실행 결과가 다양하게 나오는 성질
완벽한 다형성을 구현하기 위해서는 상속 + 메서드 오버라이딩 + 클래스 타입변환 세 가지 개념을 합쳐야 한다.

필드 다형성
필드의 타입은 동일하지만, 대입되는 객체가 달라져서 실행 결과가 다양하게 나올 수 있는 것
