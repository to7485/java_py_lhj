제약조건
테이블에 데이터를 추가할 때(sql로 직접, 프론트에 입력된 내용으로 추가)
내가 원하지 않는 데이터가 들어오는 것을 방지하기 위해 에러를 발생시키는 것

NOT NULL : 비워두지마라
UNIQUE : 겹치지마라
PRIMARY KEY : 비우지말고, 겹치지도 마라 
CHECK : 내가 정한 값만 넣어라
DEFAULT : 값을 않넣으면 기본값을 넣겠다.
FOREIGN KEY : 관계를 맺고있는 테이블 2개에서 테이블에 반드시 존재하는 값으로 채워야 한다.
- 존재하지 않는 부서에 사원을 배치할수는 없다.
- 가입하지 않은 고객에게 주문을 넣을 수는 없다.
- 존재하지 않는 과목을  학생이 수강신청할 수 없다

테이블을 만든다 -> 데이터를 넣기 위한 준비

테이블을 생성 했으면, SQL문을 수정하는것이 불가능하다.
SQL문을 수정하고 싶으면 테이블을 삭제하고 다시만들어야 한다.

테이블의 생성
CREATE TABLE 테이블명(
    속성 타입,
    속성 타입
)

웬만하면 테이블 설계를 최대한 완벽하게 하고 작성할 것.

테이블에 데이터가 들어간 상태면 수정하기가  매우 힘들다.

alter table 테이블 명 add column

alter table 테이블명 delete column

alter table 테이블명 rename column

alter table 테이블명 modify column

alter table 테이블명 add 제약조건 종류(속성);

alter table 테이블명 drop 제약조건 종류;
