JPA(Java persistence API) 
자바 프로그램에서 데이터를 DB에 저장하기 위한 기능

자바 객체랑 관계형 데이터베이스 사이를 연결하고,
개발자가 직접 SQL을 작성하지 않고도 객체를 DB에 CRUD할 수 있도록 한다.

객체(Object)와 테이블(Relation...)을 매핑(Mapping)해준다.
이를 ORM(Ojbect-Relational Mapping)기술이라고 한다.

개발자가 따라야 할 API와 사용해야할 어노테이션을 정의한다.

모든 JPA 구현체는 표준을 따야 하므로, 특정 구현체에 종속되지 않고 어플리케이션의 이식성을 높일 수 있다.

Hibernate
JPA 인터페이스를 구현한 구현체(implements)중 하나 이다.
JPA에 정의된 모든 기능을 실제 코드로 구현하고, 이를 수행하기 위해 내부 엔진을 제공한다.

SQL 변환 : 개발자가 작성한 엔티티 상태 변경이나 JPQL을 실제 데이터베이스에 맞는 SQL 쿼리로 변환하여 실행합니다.

Dialect : 다양한 데이터베이스(MySQL, Oralce... 등) 미묘한 차이점을 처리하기 위한 방언 기능을 제공한다.

JPA -> 인터페이스 공통적인 행동에 대한 정의
Hibernate -> 구현체라면 인터페이스의 메서드를 다 구현
ㄴ JPA 표준에는 없지만 개발 편의성을 높이기 위한 고유의 기능

SQL Mapper
Mybatis
객체랑 SQL 결과를 매핑하는 기술
개발자 SQL을 직접 작성해줘야 한다.
복잡한 쿼리를 작성할 때 좋다.
SQL을 직접 작성하기 때문에 직관적으로 어떤 데이터가 올 지 알 수 있다.
