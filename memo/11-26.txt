클래스
객체 생성용 클래스(설계도)

클래스를 통해서 객체를 생성한다.

모든 객체가 가지고 있는 속성을 필드라고한다.
멤버 변수 == 필드 == 속성

실행용 클래스
main()메서드가 정의되어 있다.

생성자(constructor)
객체를 생성해주는 특수한 기능이 있는 메서드다.

모든 클래스는 생성자를 갖는다.

생성자의 정의

접근제한자 : public

접근제한자 클래스명( ){ -> 기본 생성자

}

기본생성자를 정의하지 않으면 컴파일러가 자동으로 만들어준다.

객체를 만들 때 필드에 초기값을 바로 넣고싶을 때 매개변수를 이용해
값을 받을 수 있다.

생성자 오버로딩
- 같은 이름의 생성자를 여러개 정의하는 것

Car(String model, String color) { ... }
Car(String color, String model) { ... } //오버로딩이 아님 컴파일 에러 발생

컴파일러가 생성자를 읽을 때 매개변수의 타입만 읽는다.

Car(String,String)
Car(String,String)

- 객체를 메모리에 올린다
- 객체를 만들면서 전달한 값을 필드에 초기화

static필드
static메서드

정적필드, 정적메서드
자바의 메모리 영역 중 method영역에 올라간다.
메모리에 한번만 기록이 되고
객체들은 method영역에 올라간 정적필드의 값이나 메서드를 공유한다.

main에서 실행할 때 정적필드, 메서드를 메모리에 먼저 올려놓고 시작하기
때문에 너무 많은 static은 로딩에 지연이 걸릴 수 있다.

정적필드의 사용

일반적인 필드는 객체를 생성한 후 객체를 통해 호출
정적필드는 객체 생성을 하지 않고 클래스명.필드명으로 사용이 가능하다.

main() {
변수
조건문
반복
배열
}
