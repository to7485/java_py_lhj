class SharpPencil{ //샤프펜
    private int width; //펜의 굵기
    private int amount; //남은 량
    public int getAmount(){return amount;}
    public void setAmount(int amount){this.amount = amount;}
}

class Ballpen{ //볼펜
    private int amount; //남은 량
    private String color; //볼펜의 색
    public int getAmount(){return amount;}
    public void setAmount(int amount){this.amount = amount;}
    public String getColor(){return color;}
    public void setColor(String color){this.color = color;}
}

class FountainPen{//만년필
    private int amount; 
    private String color; //볼펜의 색
    public int getAmount(){return amount;}
    public void setAmount(int amount){this.amount = amount;}
    public String getColor(){return color;}
    public void setColor(String color){this.color = color;}
    public void refill(int n){setAmount(n);}
}

상속
부모 자식 관계가 있다.
부모클래스의 멤버 변수, 메서드를 자식클래스에서 활용할 수 있다.

자식클래스가 extends 키워드를 사용해서 상속받을 클래스를 선택한다.

자식클래스에는 자식클래스만의 변수, 메서드를 정의할 수 있다.

오버라이딩
부모에 메서드를 자식 클래스의 상황에 맞게 재정의하는 것
접근제한자, 반환형, 메서드명, 매개변수는 동일하게 사용한다.
코드블럭 내의 명령만 자식의 상황에 맞게 수정

자식객체를 생성했을 때 메서드를 호출하게 되면 오버라이딩된 메서드가 호출됩니다.

오버로드 vs 오버라이드

오버로드
매개변수의 개수가 다르거나, 순서가 달라야 한다.
중복정의
상속관계가 아니어도 쓸 수 있다.

오버라이드
시그니처 부분이 다 같아야 한다.
재정의
상속관계에서만 발생한다.

super
부모 클래스를 의미한다.

super() -> 부모 생성자의 호출

super.메서드명() -> 부모 메서드의 호출
