서브쿼리가 여러개의 결과를 반환할 때
IN -> IN에 들어있는 항목중 하나랑 같은지
ANY -> 서브쿼리의 결과 중 어느 하나와 조건을 만족하냐?
컬럼 > ANY -> 서브쿼리의 결과 중 어느 하나보다라도 크냐?
컬럼 = ANY -> 서브쿼리의 결과 중 하나와 같냐
컬럼 < ANY -> 서브쿼리의 결과 중 어느 하나보다라도 작냐?
SELECT SALARY FROM EMPLOYEE;
SALARY

5000

WHERE SALARY > ANY (SELECT SALARY FROM EMPLOYEES);
SALARY가 (3000,4000,5000 중 하나보다라도 크면 TRUE)

JOIN
정규화를 통해 쪼개진 데이터들을 묶어서 보고싶을 때 사용
ON절을 통해서 JOIN을 실행하면 된다.
ON절에는 PK와 FK로 지정된 컬럼을 사용한다.
종류
INNER JOIN -> 교집합, 두 테이블에서 공통된 데이터만 조회
LEFT, RIGHT OUTER JOIN -> 두 테이블에서 공통된 데이터와 한쪽의 데이터가 조회
CROSS JOIN -> 두 테이블의 모든 경우의 수 PK와 FK로 연결된 관계아 아니어도 된다.
SELF JOIN -> 자기 자신과의 JOIN

VIEW
SELECT로 조회된 값들을 테이블로서 저장하는 가상의 테이블
복잡한 쿼리문이나 민감한 정보들을 숨길 때 VIEW를 만들어
원본테이블 대신에 사용한다.
가상의 테이블이기 때문에 INSERT, UPDATE,DELETE가 불가능하다.
원본이 바뀌면 VIEW에도 반영이 된다.
CREATE VIEW 뷰이름 AS
    SELECT문;
